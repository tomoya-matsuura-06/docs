### 0章 ネットワークの基礎知識

- 1970年代：コンピュータ同士が直接通信
- 1980年代；ウィンんドウシステム
- 1990年代：TCP/IPの登場によりネットワーク技術が普及
    - 異なるメーカーやOSのコンピュータを接軸するためには共通の通信規格が必要だった
    
- TSS（Time Sharing System）
    - ネットワークを通じてホストコンピュータを同時利用できるように
- プロトコル
    - データをどのフォーマット、対処、接続の開始と終了などのルール（約束事）
    - メーカー独自の通信規格ばかりで互換性が低く、通信できないことがあった
- OSI参照モデル
    - プロトコル（約束事）を7つの階層に分けたガイドラインのこと
    - 階層に分けることでトラブル原因の特定が行いやすく

通信の種類

- TCP / IP
    - プロトコル群のこと。ルールの集約を指し、世界中のコンピュータがインターネットに接続するためには、このガイドラインに従う。といった事実上の標準規格のこと。
    - TCPは、コネクション型の接続で、送信側と受信側の通信路を確立（コネクション）してからデータを送る確実なデータ送信方法。
    - IPは、送信側が一方的にデータを送る。受信側との確率がない状態のため届くことが確約されていない。その分高速で軽量。webサイトで考えても、発信はしているけれど誰かに見てもらえるかわからない、片側だけの通信というイメージで考えられる。高速なのは、事前連絡もなくただ送るだけだから、軽量なのは送り主のアドレスなど最小限のデータしか付与されないから。受領番号などがないということが軽さに繋がる。
- パケット
    - 送りたいデータをネットワークで運びやすいように小さい単位に分割したもの。データの単位のこと。宛先アドレスをヘッダ、実際に送りたいデータそのものをペイロード。それまでは回線を占有してやり取りをする回線交換方式であった。
- 通信方式
    - ユニキャストが1対1での通信。ブロードキャストが不特定多数に一斉送信。マルチキャストが特定のグループに対して。エニーキャストが誰か一人に届けば良い。
- IPアドレス
    - 地域や組織の階層構造を持ったアドレス。MACアドレスがメーカー識別子と製品番号、通し番号を持っている、MACアドレスと地理的なIPアドレスを組み合わせることで端末を特定できる。MACアドレスがマイナンバーでIPアドレスが住所。

ネットワーク機器

- リピータ（第1層）
    - 電気信号を増幅させてネットワークの届く距離を伸ばす。
- ブリッジ（第2層）
    - スイッチングハブ。MACアドレスをもと関係のあるポートにだけデータを流す。
- ルータ（第3層）
    - IPアドレスをもとにパケットを転送する装置。異なるネットワーク間のデータの道案内をする。
- ロードバランサ（第4層）
    - トランスポート層以上の情報を活用してパケットを処理・転送する装置。第4層（TCP/UDP）や第7層（HTTP）で動作する。
- ゲートウェイ
    - 異なるプロトコルや形式のデータを相互変換する装置。
